# DocAnalyzer — 文档智能分析平台设计文档

## 1. 产品定位

**名称**：DocAnalyzer（基于 SuperMew）

**定位**：面向企业用户的专业文档智能分析平台，支持文档导入、智能问答、结构化分析、对比检查、报告生成。

**核心能力**：在现有 RAG + Graph RAG + Agent 架构上，新增结构化提取、文档对比、报告生成三大分析能力。

## 2. 核心功能

### 2.1 智能问答（已有）
基于上传文档的 RAG 问答，保留现有混合检索（Dense + BM25 + RRF）+ Graph RAG + Jina Rerank 能力。

### 2.2 结构化提取（新增）
从文档中自动提取实体和关键数据（当事方、金额、日期、条款、责任义务），写入 Neo4j 知识图谱，前端以卡片形式展示。

### 2.3 文档对比分析（新增）
上传两份文档，Agent 自动识别差异点、风险条款、关键变化，输出结构化对比报告。

### 2.4 分析报告生成（新增）
基于文档内容、图谱关系和对话历史，一键生成 Markdown 格式的分析报告，前端渲染并支持导出。

## 3. 用户角色

- **普通用户**：上传文档、提问、查看分析结果、生成报告
- **管理员**：管理知识库、管理用户、配置分析模板

沿用现有 RBAC 体系（admin/user），无需新增角色。

## 4. 技术架构

### 4.1 新增组件

#### Agent 工具（`backend/agent/`）

**`extract_structured_data` 工具**
- 输入：文档名称 + 提取类型（entity / clause / data）
- 实现：Milvus 检索文档相关块 → LLM 提取结构化信息 → 写入 Neo4j
- 输出：JSON 格式的结构化数据

**`compare_documents` 工具**
- 输入：文档 A 名称 + 文档 B 名称
- 实现：分别检索两份文档关键段落 → LLM 逐项对比 → 生成差异报告
- 输出：差异点列表 + 风险标记 + 变化摘要

**`generate_analysis_report` 工具**
- 输入：报告主题 + 文档范围
- 实现：综合 Milvus 检索 + Neo4j 图谱 + 对话历史 → LLM 生成 Markdown 报告
- 输出：Markdown 格式报告内容

#### 前端组件（`frontend/`）

- **分析面板**：展示结构化提取结果（实体列表、关键数据卡片）
- **对比视图**：并排展示两份文档的差异点，高亮变化
- **报告查看器**：Markdown 渲染，支持导出为 `.md` 文件

### 4.2 数据流

```
用户上传文档 → 三级分块 → Milvus + Neo4j（已有流程）
                              ↓
用户请求分析 → Agent 调用新工具 → Milvus 检索 + 图谱查询 → LLM 分析 → 结果返回前端
```

### 4.3 与现有架构的关系

- 新工具复用现有检索能力（Milvus 混合检索 + Graph RAG），不改动 `rag_pipeline.py` 核心逻辑
- 结构化提取结果写入 Neo4j，增强知识图谱，间接提升后续检索质量
- 前端新增组件独立于现有聊天界面，通过 Tab 切换

## 5. 关键流程

### 5.1 结构化提取流程

```
用户："帮我提取这份合同的关键条款"
  → Agent 判断调用 extract_structured_data
  → 从 Milvus 检索该文档全部相关块
  → LLM 提取：当事方、金额、日期、关键条款、责任义务
  → 提取结果写入 Neo4j（实体 + HAS_CLAUSE / HAS_AMOUNT 等关系）
  → JSON 返回前端 → 分析面板展示卡片
```

### 5.2 文档对比流程

```
用户："对比这两份合同的差异"
  → Agent 调用 compare_documents(doc_a, doc_b)
  → 分别检索两份文档的关键段落
  → LLM 逐项对比：条款差异、金额变化、新增/删除条款
  → 输出结构化差异报告 + 风险标记
  → 前端对比视图并排展示
```

### 5.3 报告生成流程

```
用户："生成一份关于这份合同的分析报告"
  → Agent 调用 generate_analysis_report
  → 综合：文档内容 + 图谱关系 + 对话历史中的分析结论
  → LLM 生成 Markdown 报告（概述、要点、风险、建议）
  → 前端报告查看器渲染，支持导出为 .md 文件
```

## 6. 前端设计

### 6.1 页面结构

在现有聊天界面基础上，顶部增加 Tab 导航：

- **对话**：现有聊天界面（不变）
- **分析**：结构化提取结果展示
- **对比**：文档对比视图
- **报告**：报告列表和查看器

### 6.2 分析面板

- 左侧：文档列表（可选择要分析的文档）
- 右侧：提取结果卡片（实体、关键数据、条款列表）
- 底部：对话入口，可针对提取结果继续提问

### 6.3 对比视图

- 顶部：两个文档选择器
- 主体：左右并排展示差异点，颜色标记新增（绿）/删除（红）/修改（黄）
- 侧边栏：风险提示列表

### 6.4 报告查看器

- 报告列表：按时间排序，显示关联文档
- 查看器：Markdown 渲染，支持导出为 `.md` 文件

## 7. 不做的事情（YAGNI）

以下功能不在本次范围内：

- 用户自定义分析模板配置（管理员后台配置过于复杂）
- 多语言报告生成
- 文档版本管理和变更追踪（需要额外的存储层）
- 批量文档分析
- PDF 格式导出（Markdown 足够展示）

## 8. 技术约束

- 不改动现有 RAG Pipeline 核心逻辑（`rag_pipeline.py`）
- 新工具通过 LangGraph Agent Tool 接口接入，与现有工具（天气、知识库检索）并列
- 前端保持 Vue 3 CDN 模式，不引入构建工具
- 复用现有 Milvus 检索和 Neo4j 图谱查询能力
